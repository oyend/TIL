### 2. 핵심 정리 (내 언어로)

  1. Spring Initializer에서 프로젝트를 생성
  2. bulid.gradle 파일을 갖다놓기 (원래 제대로 된 방법은 설정에서 gradle을 설정하는 것 같으나, 이전 스프링 세션에서도 그렇고 이번에도 그렇고 그래들 창이 바로 뜨지 않았음)
  3. 코끼리 버튼을 누르고 동기화

- **수동 주입과 자동 주입**
  - 수동 주입: 클래스에 어떤 객체가 생성되고 어떤 객체를 의존하는지 명시해야 함(`@Configuration`, `@Bean` 사용)
  - 자동 주입: 클래스에 어노테이션을 붙여(ex: `@Service`, `@Repository`, `@RestController`) 스캔 대상으로 만든 후 의존성이 필요한 곳에 `@Autowired`를 사용
    - 생성자가 1개인 경우 `@Autowired`를 생략해도 동작 가능
- `@GetMapping`: 클라이언트가 서버에게 특정 데이터를 달라고 요청 -> 코드 연결

### 4. 느낀 점
- 이번 PBL은 그동안의 학습 과정 중 가장 많은 오류를 일으켰다. 오류가 일어난 부분은 대부분 gradle 때문이었는데, 아마 파일들의 구조가 꼬인 것에서 문제가 발생된 것으로 추정된다.
- 처음에는 gradle을 추가하는 과정이 순탄치 않았기 때문에 gradle에 문제가 있는 것이라고 생각했으나, 지난 세션의 gradle 파일을 그대로 가져왔음에도 오류가 계속되자 gradle의 문제가 아님을 깨달았다. 
- 오류를 해결하기 위해 Dependencies 부분을 많이 수정했는데, 수정하는 과정에서 동기식 비동기식 등 어려운 말을 많이 보았다. 이는 추가적으로 조사해서 정리하는 게 좋을 것 같다.
- 어째서인지 윗 부분에 적혀있는 Spring 버전도 없는 버전으로 인식이 돼서 4.0.6 버전을 자체적으로 4.0.4 버전으로 변경했다.
- gradle 오류가 반복적으로 터지기 전에는 properties 관련 오류가 발생했다.해당 과정에서 application.yaml의 역할을 알게 되었다.
- Spring 세팅 관련으로 어려움을 느꼈을 뿐 실제로 코드를 작성할 때에는 큰 어려움이 없었다. 그래도 어노테이션 정도는 정리해서 익숙하게 만드는 편이 좋을 것 같다.
