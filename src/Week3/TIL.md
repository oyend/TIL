### 2. 핵심 정리
1. **추상 클래스**  
추상 메서드는 C++의 순수 가상 함수와 비슷한 기능 -> 추상 메서드를 생성하면 무조건 그 메서드를 자식 클래스에 구현해야 함
2. **인터페이스**  
이거는 꼭 구현해야 한다고 정책을 명시하는 계획서? 좀 더 공부 필요
3. **추상 클래스와 인터페이스의 차이점**
   - 추상 클래스는 다중 상속 불가능 / 인터페이스는 다중 상속 가능
   - 인터페이스는 가변 필드를 설정할 수 없음
   - 추상 클래스는 생성자를 가짐
4. **접근자**  
부모 클래스의 필드가 `private`일 때는 꼭 접근자를 설정해야 함(자식 클래스에서 `private` 필드에 접근할 수 없기 때문!)
5. **다형성**  
객체에 따라서 같은 메서드가 다르게 작동함
6. **코드 구성(패키지 구조)**
- 역할
    - 추상 클래스: 정책 객체를 반환하는 추상 메서드
    - 아기사자: 정책 객체 반환
    - 운영진: 정책 객체 반환
- 정책
    - 정책 인터페이스: 과제 제출 정책 표현 인터페이스(제출 가능 여부 반환 메서드)
    - 아기사자 정책 구현체: true 반환
    - 운영진 정책 구현체: false 반환
